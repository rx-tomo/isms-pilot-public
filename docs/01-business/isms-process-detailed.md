# ISMS（ISO/IEC 27001）プロセス実施ガイド

## 1. 概要

このドキュメントは、ISO/IEC 27001に基づくISMS（情報セキュリティマネジメントシステム）の初回認証取得から初期運用サイクルまでの標準的な実施プロセスを記載しています。

## 2. ISMSライフサイクルの全体像

```
準備フェーズ（3-6ヶ月）
    ↓
構築フェーズ（6-9ヶ月）
    ↓
認証審査フェーズ（2-3ヶ月）
    ↓
運用・継続的改善フェーズ（継続的）
```

## 3. フェーズ別詳細プロセス

### 3.1 準備フェーズ

#### 3.1.1 プロジェクト立ち上げ
- **経営層のコミットメント取得**
  - ISMS導入の意思決定
  - 予算・リソースの確保
  - 情報セキュリティ方針の策定

- **プロジェクト体制構築**
  - ISMS管理責任者の任命
  - ISMSプロジェクトチームの編成
  - 各部門の役割・責任の明確化

- **スコープ定義**
  - ISMS適用範囲の決定
  - 対象組織・業務・システムの特定
  - 物理的境界の明確化

#### 3.1.2 現状分析（ギャップ分析）
- **現状評価**
  - 既存のセキュリティ対策の棚卸し
  - ISO 27001要求事項とのギャップ分析
  - リスクアセスメント手法の選定

- **改善計画策定**
  - ギャップ対応計画の作成
  - 実施スケジュールの策定
  - 必要リソースの見積もり

### 3.2 構築フェーズ

#### 3.2.1 ISMS文書体系の構築
- **階層1: 方針文書**
  - 情報セキュリティ方針
  - ISMS適用宣言書

- **階層2: 規程類**
  - 情報セキュリティ管理規程
  - リスク管理規程
  - インシデント管理規程
  - 事業継続管理規程

- **階層3: 手順書・ガイドライン**
  - 各種運用手順書
  - 作業手順書
  - チェックリスト

- **階層4: 記録・様式**
  - 各種記録様式
  - 申請書フォーマット
  - 監査チェックリスト

#### 3.2.2 リスクアセスメント実施
- **リスク特定**
  - 情報資産の洗い出し
  - 脅威・脆弱性の特定
  - リスクシナリオの作成

- **リスク分析・評価**
  - リスクレベルの算定（影響度×発生可能性）
  - リスク受容基準の設定
  - リスク対応優先順位の決定

- **リスク対応**
  - リスク対応計画の策定
  - 管理策の選択（ISO 27001 附属書A）
  - 残留リスクの評価

#### 3.2.3 管理策の実装
- **技術的管理策**
  - アクセス制御
  - 暗号化
  - ネットワークセキュリティ
  - マルウェア対策

- **物理的管理策**
  - 入退室管理
  - クリーンデスク・クリーンスクリーン
  - 機器の物理的保護

- **組織的管理策**
  - 職務分離
  - 情報分類・取扱い
  - 供給者管理
  - 変更管理

#### 3.2.4 教育・訓練の実施
- **意識向上活動**
  - 全社員向けセキュリティ教育
  - 部門別専門教育
  - 新入社員教育

- **訓練・演習**
  - インシデント対応訓練
  - BCP訓練
  - 内部監査員養成

### 3.3 認証審査フェーズ

#### 3.3.1 内部監査
- **監査計画**
  - 年間監査計画の策定
  - 監査チームの編成
  - 監査基準・範囲の設定

- **監査実施**
  - 文書レビュー
  - 現地監査（インタビュー・観察）
  - 証拠収集

- **監査報告**
  - 不適合・改善機会の特定
  - 監査報告書の作成
  - 是正処置の要求

#### 3.3.2 マネジメントレビュー
- **レビューインプット**
  - 内部監査結果
  - リスクアセスメント結果
  - インシデント・不適合の状況
  - 利害関係者からのフィードバック

- **レビューアウトプット**
  - ISMS改善の機会
  - 資源配分の決定
  - 目標・方針の見直し

#### 3.3.3 認証審査対応
- **第1段階審査（文書審査）**
  - ISMS文書の提出
  - 審査員による文書レビュー
  - 改善事項への対応

- **第2段階審査（現地審査）**
  - 運用状況の確認
  - 従業員インタビュー
  - 記録・証拠の確認

- **審査後対応**
  - 不適合の是正処置
  - 改善提案への対応
  - 認証取得

### 3.4 運用・継続的改善フェーズ

#### 3.4.1 日常運用管理
- **運用監視**
  - セキュリティイベントの監視
  - KPI/メトリクスの測定
  - 定期レビューの実施

- **変更管理**
  - システム変更の影響評価
  - 文書・手順の更新
  - 関係者への周知

- **インシデント管理**
  - インシデントの検知・報告
  - 初動対応・エスカレーション
  - 原因分析・再発防止

#### 3.4.2 定期活動
- **定期レビュー**
  - 月次：運用状況レビュー
  - 四半期：リスクレビュー
  - 年次：ISMS全体レビュー

- **継続的改善**
  - 改善機会の特定
  - 改善計画の実施
  - 効果の測定・評価

#### 3.4.3 維持審査対応
- **サーベイランス審査（年1回）**
  - 部分的な適合性確認
  - 改善状況の確認
  - 有効性の評価

- **再認証審査（3年ごと）**
  - ISMS全体の包括的評価
  - 継続的改善の確認
  - 認証更新

## 4. 各プロセスで必要な成果物（サイクル別）

ISMS の運用には「A. 初回認証取得サイクル（未審査企業向け）」と「B. 維持・サーベイランスサイクル（認証済企業向け）」の 2 パターンがあります。初めて ISMS を導入する場合は A→B の順に進み、すでに認証済みの企業は B を継続的に実施します。

```
┌─────────────────────┐
│ 認証機関に登録済みか？ │
└──────────────┬──────────────┘
                    │はい                      │いいえ
                    ▼                         ▼
          B. 維持・サーベイランス          A. 初回認証取得
```

### 4.1 A. 初回認証取得サイクル（3.1〜3.3 に対応）

| フェーズ | 主な成果物 | 担当ロール例 |
| --- | --- | --- |
| 準備（3.1） | プロジェクト憲章、ISMS適用範囲定義書、ギャップ分析報告書、実施計画書 | 経営層、ISMS管理責任者 |
| 構築（3.2） | ISMS文書一式、リスクアセスメント報告書、適用宣言書（SoA）、教育・訓練記録 | 文書管理者、リスク管理者、IT/人事部門 |
| 認証審査（3.3） | 内部監査報告書、マネジメントレビュー議事録、是正処置報告書、審査対応ログ、認証書 | 内部監査員、ISMS管理責任者、経営層 |

### 4.2 B. 維持・サーベイランスサイクル（3.4 に対応）

| プロセス | 主な成果物 | 担当ロール例 |
| --- | --- | --- |
| 日常運用（3.4.1） | 運用記録・ログ、インシデント報告書、変更管理記録、KPIレポート | オペレーションチーム、CSIRT |
| 定期活動（3.4.2） | 月次/四半期レビュー資料、リスク再評価記録、改善提案書、定期レビュー議事録 | ISMS管理責任者、部門長 |
| サーベイランス審査（3.4.3） | 審査対応ログ、是正指摘対応記録、証跡集、継続改善レポート | ISMS管理責任者、関係部門 |
| 再認証審査（3年ごと） | 全体適合性評価、再認証審査報告書、再認証計画 | 経営層、ISMS管理責任者 |

> 初回認証を目指す企業は 3.1〜3.3 の内容と表 4.1 を参照後、認証取得後に表 4.2 のサイクルへ移行します。認証済みの企業は 4.2 を軸に、必要に応じて 3.4 の詳細を参照してください。

## 6. よくある課題と対策

### 6.1 文書化の負担
- **課題**: 過度な文書作成による業務負荷
- **対策**: 必要最小限の文書化、既存文書の活用

### 6.2 形骸化リスク
- **課題**: 認証取得後の運用形骸化
- **対策**: 定期的な有効性評価、インセンティブ設計

### 6.3 リソース不足
- **課題**: 専任担当者の不在、予算制約
- **対策**: 段階的導入、外部リソースの活用
