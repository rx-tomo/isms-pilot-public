# ISMS Pilot Public Docs

This directory contains a limited public documentation set for evaluating ISMS Pilot.

The docs are intended to help readers understand the product direction, local setup, current architecture, quality checks, operations notes, and UI/design conventions. They are not a complete commercial service manual, SLA, contract, or certification guarantee.

## What To Read First

- `01-business/pr-faq-workshop/pr-faq-public.md`: public PR/FAQ for the Build in Public snapshot
- `01-business/pr-faq-workshop/ai-driven-development.md`: how the AI-driven development experiment is described publicly
- `01-business/pr-faq-workshop/contribution-and-collaboration.md`: feedback, collaboration, and contribution policy
- `01-business/isms-process.md`: high-level ISMS process context
- `06-operations/development-environment-guide.md`: local development setup
- `05-quality/import-export-coverage-matrix.md`: current export/import coverage notes
- `05-quality/qa-guidelines.md`: public quality-checking guidance

## Public Documentation Boundary

This public docs set intentionally excludes internal handoff notes, private work logs, meeting notes, historical release-readiness evidence, issue export files, raw agent operating instructions, local databases, and generated artifacts.

Some implementation and planning docs describe current direction rather than a finished commercial service. When a document says a feature is a goal, direction, or future gate, it should not be read as a production commitment.
