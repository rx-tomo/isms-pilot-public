# スーパー管理者: 実装ギャップ整理

作成日: 2026-02-05

本ドキュメントは `docs/08-user-manual/04-super-admin` と実装状況を照合し、
「実装済み」「未実装（coming-soon）」「想定不足/不明」のギャップを整理したものです。

## 1. 実装済み（確認できたもの）
- テナント管理
  - 一覧取得、作成（プロビジョニング）、ロック/解除
  - 削除/復元（soft_delete/restore RPC 経由）
- ユーザー管理
  - Super Admin / System Operator 作成
- 招待管理
  - 招待一覧、受諾代行、招待削除
  - 招待の再送（API + UI 実装済み）
- 監査ログ
  - グローバル/テナントスコープの切替・閲覧
- サービス設定
  - フィーチャーフラグ表示
  - Edge Function ヘルス監視と ops-alert 受信
  - QA スモーク（`npm run e2e:super-admin`）

## 2. 未実装（マニュアル上 coming-soon）
- スーパー管理者による直接招待
- テナント横断のユーザー一覧・検索/フィルタ
- ユーザー無効化/復帰
- ロール確認・変更
- 不正アクセスの監視/アラート
- 監査ログの検索/フィルタ
- 監査ログのエクスポート
- 重要操作のアラート
- メンテナンスモード
- 通知テンプレート管理
- システムアナウンス
- セキュリティポリシー設定
- バックアップ設定

## 3. 想定不足/不明（要確認）
- テナント詳細画面でのプラン変更UIの有無（マニュアルでは変更可能と記載）
- 監査ログのエクスポート要件・優先度（coming-soon だが運用要件が未定）

## 4. 次の確認ポイント
- 仕様として「いつ・誰が・どの頻度で」実施する運用要件の明確化
- 実装ロードマップにおける優先度の再設定
