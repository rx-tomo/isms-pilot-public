# ISMS実務ワークフローモデル

## 位置づけ

初回採点では、UC-01からUC-10、既存ページ、既存QA資産の存在を主な根拠にしたため、ISMS新規導入・継続運用・SaaS運営の実務工程から必要能力を導く評価が不足していた。

この文書を、工程2「利用者別ワークフロー定義」の再評価における正本とする。

## 参照した外部基準

| 参照元 | 今回の使い方 |
| --- | --- |
| ISO/IEC 27001:2022 | ISMSの確立、運用、評価、改善の要求カテゴリを抽出する |
| ISO/IEC 27005:2022 | 情報セキュリティリスク管理の流れを抽出する |
| ISO/IEC 27017:2015 | クラウドサービス利用/提供時の管理策観点を補足する |
| ISO 19011:2018 | 内部監査の計画、実施、報告、フォローアップ観点を補足する |
| ISO 22301:2019 | 事業継続やインシデント時の運用継続観点を補足する |
| NIST CSF 2.0 | Govern/Identify/Protect/Detect/Respond/Recoverの観点で抜けを確認する |
| IPA 情報セキュリティ10大脅威 | 脅威変化、脆弱性、サプライチェーン、内部不正等の現実的リスクを補足する |
| METI サイバーセキュリティ経営ガイドライン | 経営関与、体制、説明責任、サプライチェーン観点を補足する |
| ISMS-AC 適合性評価制度 | ISMSが第三者評価を受ける制度上の前提を補足する |

## W-01 SaaS運営者の顧客設定

| 項目 | 内容 |
| --- | --- |
| 主アクター | super_admin, system_operator |
| 目的 | 顧客テナントを作成し、初期管理者、契約状態、利用権限、監査可能な運用状態を整える |
| 入力 | 顧客名、初期管理者、契約/プラン、利用開始日、必要ロール、サポート担当 |
| 作業 | テナント作成、初期管理者招待、ロール付与、契約状態登録、監査ログ確認、初期データ/テンプレート適用 |
| 出力 | 利用可能なテナント、初期ログイン可能な管理者、監査ログ、契約状態、初期設定チェック |
| 通過条件 | テナント越境なし、初期管理者がログイン可能、契約/権限状態が画面/API/ログで一致、作成・変更操作が監査ログに残る |

## W-02 顧客テナントのISMS初期導入

| 項目 | 内容 |
| --- | --- |
| 主アクター | org_admin, system_operator |
| 目的 | 新規導入組織がISMSの適用範囲、体制、資産、文書、リスク評価、管理策、タスクを初期化する |
| 入力 | 組織情報、ISMS適用範囲、組織体制、利用者、情報資産、既存規程、初期リスク、管理策候補 |
| 作業 | 組織設定、ユーザー招待、ISMSフェーズ選択、体制ロール設定、文書テンプレート作成、資産登録、リスク評価、管理策選定、タスク生成 |
| 出力 | 適用範囲、ISMS体制、初期文書、資産台帳、リスク台帳、管理策適用状況、初期タスク、未完了チェック |
| 通過条件 | 次に何をすべきか分かる、必須設定の抜けが可視化される、文書/資産/リスク/管理策/タスクが相互参照できる |

## W-03 日常・月次のISMS運用

| 項目 | 内容 |
| --- | --- |
| 主アクター | org_admin, user, approver |
| 目的 | 文書改訂、リスク見直し、タスク消化、通知、証跡記録を継続的に回す |
| 入力 | 期限、変更要求、リスク変化、インシデント、監査指摘、承認依頼 |
| 作業 | 文書更新、承認/差戻し、リスク再評価、タスク更新、通知確認、証跡添付、改善記録 |
| 出力 | 承認済み文書、リスク更新履歴、タスク完了履歴、通知履歴、監査ログ、改善記録 |
| 通過条件 | 担当者が迷わず次アクションを把握できる、期限超過や未承認が見える、重要操作が監査可能 |

## W-04 内部監査と是正

| 項目 | 内容 |
| --- | --- |
| 主アクター | auditor, org_admin, approver |
| 目的 | 内部監査の計画、実施、報告、不適合/是正、フォローアップを一連で完了する |
| 入力 | 監査範囲、監査計画、チェックリスト、証跡、対象文書、対象リスク、前回指摘 |
| 作業 | 監査計画、監査実施、証跡確認、不適合登録、是正タスク作成、報告書作成、フォローアップ |
| 出力 | 監査計画、監査記録、指摘/不適合、是正処置、報告書、フォローアップ履歴 |
| 通過条件 | 監査対象と証跡が追跡できる、不適合から是正タスクへつながる、報告書出力と履歴確認ができる |

## W-05 経営レビュー・継続的改善

| 項目 | 内容 |
| --- | --- |
| 主アクター | org_admin, approver, management reviewer |
| 目的 | ISMS運用状況を経営層が確認し、改善・資源配分・リスク対応を判断する |
| 入力 | リスク状況、監査結果、インシデント、是正状況、KPI、前回レビュー事項 |
| 作業 | 指標確認、レビュー記録作成、改善方針決定、アクション登録、次回レビュー計画 |
| 出力 | マネジメントレビュー記録、改善タスク、判断履歴、次回アクション |
| 通過条件 | 判断材料が集約される、レビュー記録が残る、改善タスクへ接続される |

## W-06 契約終了・データ保持・サポート調査

| 項目 | 内容 |
| --- | --- |
| 主アクター | super_admin, system_operator, org_admin |
| 目的 | 契約変更/終了、データ保持、エクスポート、サポート調査を安全に扱う |
| 入力 | 契約状態、解約日、保持期間、顧客依頼、調査対象ログ、エクスポート依頼 |
| 作業 | 契約状態更新、アクセス制御、データエクスポート、保持/削除方針確認、監査ログ調査 |
| 出力 | 契約状態、エクスポートファイル、調査記録、保持/削除予定、操作ログ |
| 通過条件 | 事業方針に反しない、テナント越境がない、顧客データ操作が監査可能、削除/保持の判断待ちが明示される |
