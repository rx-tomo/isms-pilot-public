# 2025-11-20 コードベースレビューと課題の集約
作成者: Antigravity / 2025-11-20

## 概要
本ドキュメントは、コードベースおよびプロジェクトドキュメントの包括的なレビュー結果をまとめたものです。バグ、不完全な実装、および検証が必要な領域を特定し、優先順位付けのために集約しています。

## 1. 重大なバグ (高優先度)

### 1.1 ISMS適用範囲設定のフォーカス喪失
- **ステータス**: **確認済み**
- **場所**: `components/settings/organization/ISMSScopeSettings.tsx`
- **問題**: `ScopeSection` コンポーネントがメインの `ISMSScopeSettings` コンポーネントの*内部*で定義されています。これにより、レンダリングのたびに再定義され、`<input>` 要素がアンマウント・再マウントされるため、キーストロークごとにフォーカスが失われます。
- **修正案**: `ScopeSection` をコンポーネントの外部に移動するか、メモ化します。

### 1.2 翻訳の問題
- **ステータス**: **確認済み**
- **場所**: `messages/en.json`
- **問題**:
    - `settings.organization.notificationChannels.emptyMessage` が欠落しており、キーがそのまま表示されてしまいます。
    - `assignments.helper` (正確なキーパスは要検証) にICU構文エラーがあるか、欠落していると報告されています。

## 2. 実装が不完全な項目

### 2.1 初期ロールウィザード
- **ステータス**: **未実装**
- **場所**: `components/settings/organization/RoleSeedWizardDialog.tsx` (欠落)
- **要件**: オンボーディング中に標準的なISMSロール（CISO、リスク管理者など）を一括登録するウィザード。
- **影響**: ユーザーが手動でロールを作成する必要があり、オンボーディングの手間が増えます。

### 2.2 設定画面のトースト通知統一
- **ステータス**: **部分的に実装済み**
- **場所**: `components/ui/ToastProvider.tsx` (存在)
- **問題**: `ToastProvider` と `useToast` フックは存在しますが、多くの設定ページ（例: `/settings/assets`, `/settings/organization`）では、依然としてアドホックなアラートや、スクロールすると消えてしまう静的なメッセージが使用されている可能性があります。
- **アクション**: 設定ページをリファクタリングし、`useToast` を使用して一貫したフィードバックを提供するようにします。

### 2.3 情報資産CSVのラウンドトリップ
- **ステータス**: **バックエンド実装済み / フロントエンド保留中**
- **場所**:
    - バックエンド: `app/api/information-assets/import/route.ts` は `mode` (insert/upsert/replace) をサポートしています。
    - フロントエンド: `/settings/assets` には、これらのモードを選択し、新しいエクスポート形式を扱うUIが必要です。
- **アクション**: 新しいインポートモードとテンプレートダウンロードをサポートするようにフロントエンドを更新します。

## 3. 検証が必要な項目

### 3.1 Dev Login プロファイル名の永続化
- **ステータス**: **修正済みの可能性が高い**
- **場所**: `app/api/dev/login/route.ts`
- **観察**: `forceProfileReset` が false の場合、`keepExistingProfile` とするロジックが含まれています。これは修正内容と一致しています。
- **アクション**: QA環境で、ログイン間でプロファイル名が実際に保持されることを検証します。

### 3.2 リスクマトリクスのドリルダウン
- **ステータス**: **実装済み**
- **場所**: `app/[locale]/risks/page.tsx`
- **観察**: `handleMatrixCellClick` 関数とクエリパラメータの処理 (`matrixImpact`, `matrixLikelihood`) がコード内に存在します。
- **アクション**: UIインタラクションが期待通りに動作することを検証します。

## 推奨事項と次のステップ

1.  **`ISMSScopeSettings` を直ちに修正する**: これはコアなセットアップに影響する深刻なUXリグレッションです。
2.  **翻訳を修正する**: `messages/en.json` および `messages/ja.json` に欠落しているキーを追加します。
3.  **初期ロールウィザードを実装する**: 新規ユーザーにとって価値の高い機能です。
4.  **情報資産CSVラウンドトリップUIを完成させる**: バックエンド機能をフロントエンドで有効にします。
5.  **Dev Login とリスクマトリクスを検証する**: これらが意図した通りに動作することを確認し、タスクをクローズします。
