# 2025-10-12 ISMS 機能再計画メモ

## 1. 背景
- Phase 1〜3 の実装方針に対して、資産管理・管理策ライブラリなどの主要機能は実装済みだが、計画書のステータスが未更新のままとなっている。
- 今後の開発スコープを明確化し、優先度・フェーズを再整理したうえで、四半期ベースのロードマップを提示する。

## 2. 現状サマリー（未完了／要調整機能）
| カテゴリ | 未対応項目 | 備考 |
| --- | --- | --- |
| 文書管理 | Word/PDF エクスポートの最終 QA、テンプレートダウンロード導線 | 実装済み API あり。運用ドキュメント整理が必要 |
| リスク管理 | Excel エクスポート、Annex A 管理策連携の高度化 | Annex A ライブラリは実装済み、新たなレポート要件は未着手 |
| ギャップ分析 | 進捗ダッシュボード高度化、CSV 自動配信 | `/risks/gap-analysis` は提供済み。スケジューラ連携が未実装 |
| タスク管理 | ダッシュボードカンバン、リアルタイム更新 | 既存 UI は一覧中心。通知連携と SLA 管理が未完了 |
| 内部監査 | 監査報告書出力（PDF）、Dev Login Seeds 拡張 | 内部監査 UI は完了、出力フォーマット調整が必要 |
| インシデント管理 | モジュール未着手 | Phase 2 後半に着手予定 |
| 教育・訓練 | 受講管理モジュール未着手 | Phase 3 領域、スコープ再確認が必要 |
| KPI / レポート | ダッシュボード、マネジメントレビュー | KPI 定義とデータソース整理から着手 |

## 3. 主要リスク
- スコープ膨張：Phase 2 以降で未定義のタスクが多いため、四半期ごとのリリースゴールを先に固定する。
- リソース配分：インシデント管理や教育モジュールは新規開発が多く、専門ドメインヒアリングが必要。
- 品質保証：エクスポート系は外部監査に直結するため、QA のサイクルを早期に組み込む。

## 4. 次ステップ概要
1. Phase 2 を「リスク&監査拡張」「運用基盤」「レポート整備」の3ストリームに再編。
2. Phase 3 を「教育・定着」「継続改善」「高度レポート」の3テーマに整理。
3. 各ストリームごとに MVP ゴールと依存関係を明示し、四半期別マイルストーンを設定。

## 5. ロードマップ概観（2025Q4〜2026Q2）
| 期 | ストリーム | マイルストーン |
| --- | --- | --- |
| 2025Q4 | リスク&監査拡張 | リスク Excel エクスポート、監査報告書 PDF、自動チェックリスト生成完了 |
| 2025Q4 | 運用基盤 | タスクダッシュボード拡張（カンバン/フィルター）と通知連携 |
| 2026Q1 | レポート整備 | ギャップ分析定期レポート配信、ダッシュボード KPI MVP |
| 2026Q1 | 教育・定着 | 教育計画/受講管理の基盤リリース |
| 2026Q2 | 継続改善 | インシデント管理 MVP、是正予防処置テンプレート連携 |
| 2026Q2 | 高度レポート | マネジメントレビュー資料テンプレート、PDF/Excel 一括出力 |

## 6. 依存関係メモ
- 文書エクスポート系は Phase 1 の QA で完了予定。レポート整備ストリームの前提。
- リスク Excel エクスポートは Annex A 管理策ライブラリとの連携が必要。
- 教育・定着ストリームはユーザー／権限管理の安定化を前提とする。
