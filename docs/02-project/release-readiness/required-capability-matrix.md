# ISMS必要能力マトリクス

## 位置づけ

この文書は、工程3「機能充足ギャップ分析」の正本である。現行実装に何があるかではなく、ISMS実務ワークフローを成立させるために製品が備えるべき能力を先に定義する。

2026-05-14の後続Goal 1で、`isms-practical-research.md` を追加した。Goal 2以降は、CAP-01〜CAP-24に加えてCAP-25〜CAP-32追加候補もFit & Gap棚卸し対象に含める。

## 既存CAPの補正メモ

| 対象 | Goal 2での見方 |
| --- | --- |
| CAP-04 | 利害関係者、順守義務、適用範囲、除外理由、境界システムも含めて確認する。 |
| CAP-09 | リスク評価/対応だけでなく、リスク基準、残留リスク承認、コミュニケーション、監視レビューを確認する。 |
| CAP-10 | 管理策選定だけでなく、SoA、運用頻度、証跡、例外、レビュー、関連リスクを確認する。 |
| CAP-11/CAP-14 | CAP-11は汎用タスク基盤、CAP-14は監査不適合/CAPAとして分ける。 |
| CAP-13 | 監査プログラム、独立性、監査員力量、監査証拠、フォローアップ確認を含める。 |
| CAP-15 | 経営レビューの入力、判断、資源配分、リスク受容、改善指示を確認する。 |
| CAP-16 | インシデント、変更管理、復旧/BCPの分割を検討する。 |
| CAP-18 | 監査提出パッケージ、証跡束、契約終了時ポータビリティとして確認する。 |

## 採点ルール

| 判定 | 意味 |
| --- | --- |
| 実装済み | コード、画面/API、テストまたはQA証跡で確認できる |
| docsのみ | 仕様や手順はあるが、現行動作の証跡がない |
| 未検証 | 実装資産はあるが、今回または最新状態で動作確認できていない |
| 不足 | 必要能力に対して実装または運用手順が不足している |
| 判断待ち | データ削除、料金、法務表現など事業判断が先に必要 |
