# Security Policy

Please do not report security issues in public GitHub issues.

If you believe you have found a vulnerability, contact the repository owner through a private channel or GitHub Security Advisories if enabled for this repository. Include a concise description, reproduction steps, affected files or routes, and any relevant logs with secrets redacted.

Do not include real credentials, customer data, access tokens, cookies, private keys, or production database contents in a report.

This public snapshot is intended for evaluation. Production use requires a separate security review and commercial permission.
