# ISMS実務・規格・市場ノウハウ調査

## 位置づけ

この文書は後続Goal 1の成果物である。現行実装や既存ページの有無から逆算せず、ISMS新規導入、継続運用、内部監査、マネジメントレビュー、SaaS運営に必要な業務と能力を再定義する。

実装には入らず、Goal 2のFit & Gap棚卸しで使う調査正本として扱う。

## 参照した公開情報

| 参照元 | 今回の使い方 |
| --- | --- |
| ISO/IEC 27001:2022 | ISMSの要求事項、適用範囲、リスク管理、運用、評価、改善を整理する基準。 |
| ISO/IEC 27002:2022 | Annex A管理策の実装ガイダンス、SoAと管理策運用証跡の粒度を考える基準。 |
| ISO/IEC 27005:2022 | 情報セキュリティリスクの識別、分析、評価、対応、コミュニケーション、監視レビューを整理する基準。 |

| 業務観点 | 主な内容 | 既存Wへの反映 |
| --- | --- | --- |
| 組織状況・利害関係者・適用範囲 | 組織状況、利害関係者、順守義務、適用範囲、除外理由、境界システム。 | W-02を拡張 |
| リスク基準とリスク管理サイクル | リスク基準、受容基準、識別、分析、評価、対応、残留リスク、監視レビュー、コミュニケーション。 | W-02/W-03を拡張 |
| SoAと管理策運用 | Annex A管理策の適用/不適用、理由、実装状態、運用頻度、証跡、例外、レビュー。 | W-02/W-03を拡張 |
| 教育・力量・認識 | 役割別教育、内部監査員等の力量、受講記録、認識確認。 | W-02/W-03/W-04へ追加 |
| 監視測定・KPI・目的管理 | 情報セキュリティ目的、KPI、測定結果、レビュー、改善への接続。 | W-03/W-05を拡張 |
| 内部監査プログラム | 監査プログラム、独立性、監査員力量、サンプリング、監査証拠、フォローアップ。 | W-04を拡張 |
| 経営レビューと資源配分 | 入力情報、リスク受容、資源配分、方針/目的見直し、改善指示。 | W-05を拡張 |
| インシデント・変更・復旧 | インシデント記録、証拠保全、変更影響、バックアップ、復旧、再発防止。 | W-03/W-05を拡張 |
| 供給者・クラウド・サプライチェーン | 委託先、クラウド、契約要求、定期レビュー、サプライチェーンリスク。 | W-02/W-03/W-05へ追加 |

| SaaS運営者の責任境界 | テナント分離、管理者アクセス、契約終了、保持/削除、サポート調査、運営者自身のセキュリティ。 | W-01/W-06を拡張 |

## 既存CAPの補正方針

| 対象 | 補正方針 |
| --- | --- |
| CAP-04 ISMSスコープ・組織体制設定 | 利害関係者、順守義務、適用範囲、除外理由、境界システムまで含める。 |
| CAP-09 リスク評価・リスク対応 | リスク基準、受容基準、残留リスク承認、コミュニケーション、監視レビューを明示する。必要ならGoal 2で分割する。 |
| CAP-10 管理策・SoA支援 | 管理策選定だけでなく、運用頻度、証跡、例外、レビュー、関連リスクまで含める。 |
| CAP-11 タスク・是正管理 | 汎用タスク基盤とする。監査不適合/CAPAはCAP-14に寄せる。 |
| CAP-13 内部監査 | 監査プログラム、監査員力量、独立性、監査証拠、フォローアップ確認を含める。 |
| CAP-14 不適合・是正フォローアップ | CAPAとして原因分析、是正、効果確認、再発防止まで含める。 |
| CAP-15 マネジメントレビュー | レビュー入力、判断、資源配分、リスク受容、改善指示を必須寄りに引き上げる。 |
| CAP-16 インシデント/変更管理 | インシデント、変更、復旧/BCPをGoal 2で分割検討する。 |
| CAP-18 エクスポート/ポータビリティ | 監査提出パッケージ、証跡束、契約終了時ポータビリティへ役割を広げる。 |
| CAP-21 AI支援の人手確認・監査ログ | MVPではAI高度化ではなく、人手確認、採否、責任境界、ログを最低条件とする。 |
| CAP-22 データ保持・論理削除・契約終了運用 | 事業判断待ち。Goal 2で実装探索はできるが、最終仕様確定にはPO判断が必要。 |
| CAP-23 セキュリティ検証・脆弱性管理 | 製品自身のセキュリティ保証と、顧客ISMSの脆弱性管理支援を混同しない。 |
| CAP-24 品質検証・主要ジャーニー確認 | リリース検証の証跡であり、製品機能の監査証跡とは分けて扱う。 |

| ID | 必要能力 | 主なワークフロー | MVP判断 | 期待される証跡 |
| --- | --- | --- | --- | --- |
| CAP-25 | 教育・力量・認識管理 | W-02, W-03, W-04 | 重要 | 教育計画、受講記録、力量記録、認識確認、内部監査員力量 |
| CAP-26 | 供給者/クラウド/サプライチェーン管理 | W-02, W-03, W-05 | 重要 | 委託先台帳、契約要求、評価記録、定期レビュー、リスク対応 |
| CAP-27 | 情報セキュリティ目的・KPI・監視測定 | W-03, W-05 | 重要 | 目的、KPI、測定結果、レビュー、改善アクション |
| CAP-28 | リスク基準・残留リスク受容 | W-02, W-03, W-05 | 必須 | リスク基準、受容基準、残留リスク、承認者、レビュー履歴 |
| CAP-29 | 管理策運用証跡・Evidence Vault | W-03, W-04, W-05 | 必須 | 証跡ファイル/リンク、取得日、対象期間、所有者、承認、関連リスク/管理策 |

## MVPで外せない証跡

Goal 2以降のFit & Gapでは、画面/API/DB/テストの存在だけでなく、以下を実務証跡として確認する。

- ISMS適用範囲、利害関係者、順守義務、除外理由、境界システム。
- 情報セキュリティ方針、役割責任、承認履歴。
- リスク評価手順、リスク登録簿、リスク対応計画、残留リスク承認。
- SoA: Annex A管理策ごとの適用/不適用、理由、実装状態、関連リスク、運用証跡。
- 教育、力量、認識の記録。
- アクセスレビュー、脆弱性管理、バックアップ、ログ監視、変更管理、インシデントの主要運用証跡。
- 内部監査計画、監査報告、監査証拠、所見、不適合、フォローアップ確認。
- マネジメントレビュー議事録、入力情報、決定、資源配分、アクション。
- 是正処置、原因分析、効果確認。
- 委託先/クラウド評価、契約要求、定期レビュー。
- 審査提出または説明に使える証跡パッケージ。

## Goal 2へ進むための判定

Goal 2へ進める。ただし、Goal 2はCAP-01〜CAP-24だけでなく、本書のCAP-25〜CAP-32追加候補と既存CAP補正方針を正本として読む必要がある。

追加調査が必要なもの:

- ISO/IEC 27002:2022 Annex A 93管理策の詳細マッピング。
- JIS Q 27001:2025およびISO/IEC 27001:2022/Amd 1:2024の国内表記・気候変動追補の扱い。
- 国内競合SaaS/コンサル市場で、MVPに求められる証跡パッケージの粒度。

事業判断待ち:

- 本プロダクトを「認証取得支援」と表現するか、「ISMS運用・証跡管理支援」に留めるか。
- 審査提出パッケージの保証範囲。
- 契約終了時の保持、削除、復旧、サポート調査の責任境界。
- 供給者管理、教育管理、BCP/復旧をMVP必須に含める深さ。
