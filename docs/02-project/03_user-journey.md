# ISO/IEC 27001 初回認証取得までのユーザージャーニー

このドキュメントは、本ISMSマネジメントシステムを使用して、組織がISO/IEC 27001認証を初めて取得するまでのユーザージャーニーを詳細に記載したものです。

## 概要

- **認証取得期間**: 約6-7ヶ月
- **主要フェーズ**: 7つのフェーズ
- **各ステップ**: 誰が、どの機能を使用するかを明記

## フェーズ詳細

### 1. 初期セットアップフェーズ（1-2週目）

#### 1.1 アカウント作成とトライアル開始
- **実行者**: IT部門責任者または経営者
- **使用機能**: サインアップ画面（`/auth/signup`）
- **実施内容**: 
  - 会社情報を入力してアカウント作成
  - 14日間の無料トライアルを開始
  - メール認証を完了

#### 1.2 組織設定
- **実行者**: 組織管理者（Organization Admin）
- **使用機能**: 組織設定画面（`/settings/organization`）
- **実施内容**:
  - 組織の基本情報を設定
  - ISMSの適用範囲を定義
  - 部門情報を登録

#### 1.3 ユーザー招待
- **実行者**: 組織管理者
- **使用機能**: ユーザー管理画面（`/settings/users`）
- **実施内容**:
  - ISMS推進チームメンバーを招待
  - 役割を割り当て（User、Auditor、Approver）
  - アクセス権限を設定（メール招待と細粒度権限 UI を用いてモジュール単位のアクセスを調整）

### 2. 準備フェーズ（3-8週目）

#### 2.1 文書体系の構築
- **実行者**: ISMS推進担当者
- **使用機能**: 
  - 文書管理（`/documents`）
  - テンプレート（`/documents/templates`）
- **実施内容**:
  - ISMS基本方針書を作成
  - 情報セキュリティポリシーを作成
  - 各種規程・手順書をテンプレートから作成（※テンプレート自動展開と新規文書フォームの API 連携は OR-03/04 で実装予定）
  - フォルダ構造を整理

#### 2.2 文書の承認
- **実行者**: 承認者（Approver）
- **使用機能**: 文書管理の承認ワークフロー
- **実施内容**:
  - 作成された文書をレビュー
  - コメントでフィードバック
  - 承認または差し戻し

#### 2.3 リスクアセスメント実施
- **実行者**: 各部門の責任者とISMS推進担当者
- **使用機能**: 
  - リスク管理（`/risks`）
  - 新規リスク作成（`/risks/new`）
- **実施内容**:
  - 情報資産の洗い出し
  - リスクの特定（情報セキュリティ、事業継続性、コンプライアンス等）
  - 影響度と発生可能性を評価（5×5マトリクス）
  - リスク対応計画を策定（回避、低減、移転、受容）
  - 規格要求とのギャップを整理し、Annex A 管理策を割り当てる（※情報資産台帳、ギャップ分析、管理策ライブラリ、レポート出力は OR-06〜OR-09 で実装予定）

### 3. 実装フェーズ（9-16週目）

#### 3.1 タスク管理による実装
- **実行者**: ISMS推進担当者とタスク実行者
- **使用機能**: 
  - タスク管理（`/tasks`）
  - 新規タスク作成（`/tasks/new`）
- **実施内容**:
  - リスク対応計画に基づくタスクを作成
  - 担当者を割り当て、期限を設定
  - 進捗状況を定期的に更新
  - 必要な証跡を添付

#### 3.2 進捗モニタリング
- **実行者**: ISMS責任者
- **使用機能**: ダッシュボード（`/home`）
- **実施内容**:
  - 全体の進捗状況を確認
  - 遅延タスクを特定し、フォローアップ
  - KPIを確認（文書承認率、タスク完了率等）

### 4. 内部監査フェーズ（17-20週目）

#### 4.1 監査計画作成
- **実行者**: 内部監査責任者（Auditor）
- **使用機能**: 
  - 監査管理（`/audit`）
  - 監査計画作成（`/audit/plans/new`）
- **実施内容**:
  - 内部監査計画を作成
  - ISO 27001:2022要求事項チェックリストを準備
  - 監査スケジュールを設定

#### 4.2 内部監査実施
- **実行者**: 内部監査員（Auditor）
- **使用機能**: 監査チェックリスト機能
- **実施内容**:
  - 各要求事項の適合性を評価
  - 不適合事項を記録
  - 改善の機会を特定
  - 監査証跡を添付

#### 4.3 是正処置
- **実行者**: 各部門責任者
- **使用機能**: タスク管理、リスク管理
- **実施内容**:
  - 不適合事項に対する是正処置計画を作成
  - タスクとして登録し、実行
  - 完了後、証跡を添付

### 5. 認証審査準備フェーズ（21-24週目）

#### 5.1 文書の最終確認
- **実行者**: ISMS推進チーム全員
- **使用機能**: 文書管理
- **実施内容**:
  - すべての必須文書が最新版であることを確認
  - 承認済みステータスの確認
  - 審査員向け文書一覧を準備

#### 5.2 審査対応準備
- **実行者**: ISMS責任者
- **使用機能**: 監査管理、文書管理
- **実施内容**:
  - 外部審査用の監査計画を作成
  - 必要な証跡を整理
  - 審査員向けアクセス権限を準備

### 6. 認証審査フェーズ（25-28週目）

#### 6.1 第1段階審査（文書審査）
- **実行者**: ISMS責任者と文書管理担当者
- **使用機能**: 文書管理、監査管理
- **実施内容**:
  - 審査員に文書へのアクセスを提供
  - 質問事項への回答を記録
  - 指摘事項をシステムに登録

#### 6.2 第2段階審査（現地審査）
- **実行者**: 全関係者
- **使用機能**: すべての機能
- **実施内容**:
  - システムを使った運用実績を提示
  - リスクアセスメント結果を説明
  - タスク管理による改善活動を実証
  - 内部監査の記録を提示

### 7. 認証取得後の運用フェーズ

#### 7.1 継続的改善
- **実行者**: ISMS推進チーム
- **使用機能**: 
  - 通知機能（`/notifications`）
  - ダッシュボード
- **実施内容**:
  - 定期的なレビュー実施の通知を受信
  - KPIをモニタリング
  - 改善機会を特定し、タスク化

#### 7.2 サブスクリプション管理
- **実行者**: 組織管理者
- **使用機能**: サブスクリプション設定（`/settings/subscription`）
- **実施内容**:
  - トライアルから有料プランへ移行
  - ユーザー数や容量に応じてプラン選択
  - 請求情報を管理

## 8. EARS 要件（ユーザージャーニー）
1. **When** System Operator がオンボーディングチェックリストの各ステップを完了したとき、**the system shall** 組織設定・ユーザー招待・適用範囲登録の進捗をリアルタイムで更新し、未完ステップには具体的な次アクションを提示する。
2. **When** 承認者が文書ワークフローで承認または差し戻しを実行したとき、**the system shall** ジャーニー上の「文書体系構築」フェーズにそのイベントを反映し、次に行うべき監査準備タスクを通知する。
3. **When** リスクアセスメントでハイリスクが検出されたとき、**the system shall** リスク対応タスクを自動生成し、ユーザージャーニーの「実装フェーズ」にアラートを表示して関係者へフォローアップを促す。
4. **When** 内部監査が完了して是正処置が登録されたとき、**the system shall** 「認証審査準備フェーズ」を自動的に解放し、審査対応用のチェックリストとレポート生成リンクを表示する。

## 機能開発への示唆

このユーザージャーニーに基づいて、以下の機能開発を優先的に進めることを推奨します：

1. **基本機能の充実**
   - 文書管理システムの高度化（バージョン管理、承認ワークフロー）
   - リスクアセスメント機能の強化（リスクマトリクス可視化）
   - タスク管理とプロジェクト管理機能の統合

2. **監査対応機能**
   - ISO 27001:2022チェックリストテンプレート
   - 監査証跡の自動収集・整理機能
   - 不適合管理と是正処置追跡機能

3. **ダッシュボードとレポート**
   - 経営層向けダッシュボード
   - 進捗レポートの自動生成
   - KPI/KRIの可視化

4. **外部連携**
   - 審査機関向けゲストアクセス機能
   - Word/Excel形式でのデータエクスポート
   - 既存システムとのAPI連携
