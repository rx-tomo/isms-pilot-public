## いちばん重要な問題の説明

最も重要なのは、監査レポートPDF export APIです。

監査レポートは、組織名、監査計画、監査人、チェックリスト、不適合、証跡のような機微な情報を含みます。このAPIがログイン確認やテナント確認なしで呼べると、URLやIDが漏れたときに、別の人が監査情報を取れてしまう可能性があります。

この1件だけで、採点基準上は最終ランクが最大Dになります。点数が98点でもDになるのは、このルールのためです。

## 修正優先度

1. 監査レポートexport APIに認証、認可、テナント境界チェックを追加する。
2. 通知配送APIに認証、認可、冪等性、再送制御を追加する。
3. `BETTER_AUTH_SECRET` と storage signing key を必須化し、未設定なら安全に失敗させる。
4. client componentからserver serviceを直接呼ばない構造へ直す。API routeまたはserver component境界に寄せる。
5. 動的SQLの列名をallowlist化する。
6. lint警告、`any`、巨大client pageを段階的に整理する。
