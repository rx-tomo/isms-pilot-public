# Riscala AI for ISMS 開発ロードマップ

## 概要

ユーザージャーニーに基づき、ISO 27001認証取得の流れに沿った機能開発を段階的に実施します。開発効率を最大化するため、基盤となる共通機能から開発し、各フェーズで必要な機能を順次追加していきます。

## 開発方針

### 基本原則
1. **ユーザージャーニー優先**: 認証取得フローに沿った機能から実装
2. **共通基盤先行**: 認証・権限管理などの基盤機能を最初に構築
3. **MVP段階リリース**: 3ヶ月で最小限の価値を提供
4. **段階的拡張**: ユーザーフィードバックを反映しながら機能追加
5. **技術的負債の最小化**: 適切なアーキテクチャ設計と継続的リファクタリング

### 技術スタック（確定）
- **Frontend**: Next.js 14.2.20 + TypeScript + Tailwind CSS
- **Backend**: Better Auth + Drizzle ORM (SQLite) — Supabase 完全削除済み
- **ORM**: Drizzle ORM (SQLite, libsql)
- **Payment**: Stripe
- **Hosting**: Vercel
- **i18n**: next-intl

## 技術的考慮事項

### データベース設計の優先事項
1. **マルチテナント対応**: 全テーブルでorganization_idによる分離
2. **監査ログ**: 全ての変更操作を記録
3. **ソフトデリート**: データの論理削除を実装
4. **パフォーマンス**: 適切なインデックス設計

### セキュリティ実装
1. **Row Level Security**: 全テーブルで実装
2. **API Rate Limiting**: DDoS対策
3. **入力検証**: XSS/SQLインジェクション対策
4. **暗号化**: 機密データの暗号化

### スケーラビリティ
1. **キャッシュ戦略**: Redis導入検討
2. **CDN活用**: 静的リソースの配信最適化
3. **非同期処理**: 重い処理のバックグラウンド化
4. **データベース最適化**: クエリチューニング
