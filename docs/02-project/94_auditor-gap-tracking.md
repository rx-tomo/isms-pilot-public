# 監査員（auditor）ギャップ整理・追加設計（内部監査前提）

## 前提
- 本システムで想定する監査は **社内で実施するISMS内部監査** のみ
- 外部監査/第三者評価は対象外（文言とフローを内部監査に統一）

## 追加要件（実装計画に反映）

### 監査員の登録フロー
- 組織管理者が内部監査委員（auditor）を招待し、ロール付与する
- 招待された監査員は監査トップから計画作成に着手

### 監査計画の入力項目拡張
- 監査対象の**組織選択**
- 監査員サイン（オンライン上の承認/署名）
- 監査実施日の明確化（既存日程項目に加え、実施日の確定）

### 監査報告書の承認ワークフロー
- 承認者は「情報セキュリティ責任者」
- 組織設定の体制ロール（ISMS管理責任者等）から承認者を設定
- 監査報告書は承認ステータスを持ち、承認/差戻しが可能

### フォローアップの進捗入力
- 被監査部門の担当者がアプリ上で進捗入力
- 監査員は「フォローアップ状況ダッシュボード」で進捗を確認

### 監査関連のCSVシード取り込み
- 監査対象組織/担当者/監査員の基礎データをCSVで取り込み
- 取り込んだデータは監査計画の選択肢に反映
- 既存のCSV取込体験と整合させる

## 実装済み（確認済み）
- 監査計画の新規作成/詳細編集/チーム編成
- チェックリストの記録/証跡アップロード/不適合登録
- 不適合管理の是正処置登録と進捗管理
- 監査報告書の作成/更新/PDF出力
- 監査報告書一覧の検索/フィルタ/ダウンロード
- 要求事項の適用可否とチェックリスト一括生成
- 監査計画の監査対象ユニット（audit_units テーブル + 監査計画への反映）
- 監査員サイン（audit_plans に auditor_signature カラム追加）
- 監査報告書の承認WF（approval_status + 共通承認ワークフロー連携）
- フォローアップ進捗入力のUI（follow_up_records テーブル + CRUD + UI）
- 監査関連CSVシード取り込み（audit_units import API + setup UI）
