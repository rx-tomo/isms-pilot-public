# 2025-11-18 Phase4 未実装領域 EARS 要件
作成者: Codex / 2025-11-18

## 1. 教育・訓練モジュール
1. **When** 管理者が教育計画を作成したとき、**the system shall** 研修名・対象ロール・実施期間・必須/任意フラグを保存し、対象ユーザーへ通知タスクを自動生成する。
2. **When** 受講者が研修完了フォームで「完了」を提出したとき、**the system shall** 受講ログに日時と评分を記録し、未完了ユーザーのリマインダーキューから除外する。
3. **When** 研修の締切を迎えたとき、**the system shall** 未完了ユーザー一覧と遅延日数を含むレポートをダウンロード可能な CSV/PDF として生成する。

## 2. インシデント管理
1. **When** ユーザーがインシデントを登録したとき、**the system shall** 重大度・影響範囲・発生日を必須項目として収集し、選択した責任者に通知を送る。
2. **When** インシデントの重大度が「高」または「重大」に変更されたとき、**the system shall** タスクモジュールへ是正処置テンプレートを自動作成し、実施期限を 30 日以内に設定する。
3. **When** インシデントがクローズされたとき、**the system shall** 経過時間と対応ステップを監査ログへ記録し、レポート用に CSV/PDF へエクスポートできるようにする。

## 3. KPI / マネジメントレビュー
1. **When** 組織設定で KPI ダッシュボードが有効化されているとき、**the system shall** リスク/監査/タスク/教育の主要指標（例: 未対応リスク件数、監査完了率）を 24 時間ごとに再計算し、ダッシュボードに表示する。
2. **When** マネジメントレビューの議事録を作成したとき、**the system shall** 会議参加者、アジェンダ、決定事項、アクションアイテムをテンプレート化された Word/PDF に出力し、Storage に保存する。
3. **When** KPI 指標が事前定義された閾値を下回ったとき、**the system shall** アラートを生成し、対象ロール（例: CISO, Org Admin）へ通知するとともに、次回レビューの議題候補に自動追加する。
