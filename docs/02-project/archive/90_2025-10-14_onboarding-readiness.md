# 2025-10-14 Org Admin Onboarding Readiness ギャップ整理

## 背景
UC-01 "初期オンボーディング" をウォーキングスケルトンで検証した結果、Org Admin が無料トライアル期間中に体制構築と文書整備を完了し、部署メンバーへ初回リスクアセスメントを依頼できる状態に達していない。以下のギャップをタスクレベルで明文化し、Sprint "Onboarding Readiness" に取り込む。

## ギャップ一覧
1. **招待メール送信が未実装**
   - `UserService.inviteUser` がトークン生成後にログ出力のみ。
   - Supabase Edge Functions / メール API 連携が未着手。
   - ロードマップに「メール送信未実装」と記載あり。
2. **細粒度権限 UI が存在しない**
   - 招待時にロールを選択するのみで、モジュール単位の権限制御不可。
   - RBAC モデルの拡張と UI 追加が必要。
3. **文書登録フローがダミー実装**
   - `/documents/new` はコンソール出力後に一覧へ遷移するだけ。
   - DocumentService や `document_versions` と連携していない。
   - 2025-10-16 OR-03 で解消（Markdown 本文保存・初期バージョン生成まで完了）。
4. **テンプレートからの文書生成が未完成**
   - `DocumentService.createFromTemplate` が TODO のまま。（2025-10-15 完了済み。テンプレート本文のファイル生成とバージョン作成を実装）
   - Storage/`document_versions` へのファイル登録が欠落。（2025-10-15 付けで文書作成時に自動保存されるよう更新）
5. **プロジェクト体制管理 UI がない**
   - ISMS 体制図（責任者・各部門担当）を登録する機能が未実装。
6. **情報資産台帳が存在しない**（OR-06 対応済み）
   - 資産棚卸しがシステム内で完結せず、リスク登録と連携できない。現在は情報資産 CRUD とリスク紐付けが利用可能。
7. **ギャップ分析レポートが未実装**
   - 規格要求への適合状況を部署別に把握する UI/帳票が未整備。
8. **ISO Annex A 管理策ライブラリが未整備**
   - リスク対応策の選択肢がシステムに存在しない。
9. **文書・リスクのエクスポートが未実装**
   - Word/PDF/Excel で成果物を出力できず、審査準備が滞る。
